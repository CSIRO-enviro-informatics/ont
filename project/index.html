This document describes a core ontology for the description of a project, aimed at supporting linked data publishing of project descriptions across a number of domains. A project is a time-bounded activity involving one or more participants playing various roles, using inputs and materials and generating products of some form. The ontology is designed to capture the most common features of a project descxription, with an expectation that it will be specialized for applications that have more detailed requirements.

This is an initial draft produced for consideration by the W3C Data eXchange Working Group.

Introduction

This document describes a core ontology for the description of a project. The project is a key unit of activity in modern business. A project has specified start and end dates, involving participants playing various roles such as leader, sponsor, funder, owner, and generating products of some form. Projects provide key context for most productive activities and their outputs, including datasets.

A number of previous initiatives have developed project ontologies. ... FOAF, DOAP, VIVO-ISF, dbPedia ...

Overview of project ontology

This ontology is designed to enable publication of information describing projects, including research projects. It is not designed to support project management, though there is support for sub-activities. The ontology has a dependency on W3C PROV-O, and minor dependencies on some other vocabularies and ontologies, but as far as possible is intended to be domain-neutral. The expectation is that domains and applications will specialize or extend this ontology for more specific purposes.
